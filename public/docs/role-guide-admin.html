System Administrator Guide

You have full system control. This guide covers user management, module access configuration, system settings, email templates, master data, and all administrative tasks.

Modules You Can Access

As an Admin, you have access to every module in the system. The sidebar shows all items without filtering. Key administration modules are highlighted below:

Administration > Master Settings
Manage departments, categories, sub-categories, vendors, branches, locations, and approval thresholds.
Administration > Module Access
Configure which modules and sub-items each role can access.
Administration > System Config
Tenant-level configuration: org name, logo, currency, date format, SMTP settings.
Administration > Email Templates
Create and edit email templates for notifications (ticket raised, WO completed, etc.).
Administration > Inventory Master
Manage inventory item catalog: add items, set categories, define units.
Administration > SLA Matrix
Define response and resolution SLA timelines by priority and category.
Administration > Escalation Matrix
Configure escalation levels, timings, and notification recipients.
Administration > Support Matrix
Map support teams to asset categories and locations.
Administration > User Activity
Monitor user login activity and session history.
Administration > Audit Trail
View complete audit trail of all system actions.
Administration > Notifications
View and manage system-wide notifications.
Administration > Notification Preferences
Configure notification channels and preferences per event type.

Plus All Operational Modules

You also have full access to: Dashboard, My Assets, Asset Master (all sub-items), Asset Indent, Store & Inventory, Maintenance (all sub-items), Contracts & Coverage (all sub-items), Finance & Analytics (all sub-items), and Operations (all sub-items).

Step-by-Step Workflows

Workflow 1: How to Create a New User

  1. Navigate to Administration > Master Settings in the sidebar.
  2. Go to the Users section.
  3. Click Add User.
  4. Fill in: Name, Email, Employee ID, Phone Number.
  5. Select the Role (Executive, Supervisor, HOD, Operations, Finance, CEO_COO, or Admin).
  6. Select the Department and Branch/Location.
  7. Set an initial Password (the user will be prompted to change it on first login).
  8. Click Save to create the user account.
  9. The user receives a welcome email with login credentials (if SMTP is configured).

Workflow 2: How to Configure Module Access for a Role

  1. Navigate to Administration > Module Access in the sidebar.
  2. Select the Role you want to configure from the role dropdown.
  3. The screen displays all available modules and sub-items as a checklist.
  4. Check or uncheck modules to grant or revoke access for that role.
  5. For modules with sub-items (e.g., Asset Master), expand to configure individual sub-item access.
  6. Click Save to apply the changes.
  7. Users with this role will see the updated sidebar on their next login or page refresh.

Workflow 3: How to Set Up Email Templates

  1. Navigate to Administration > Email Templates in the sidebar.
  2. View the list of system events that trigger emails (e.g., Ticket Raised, WO Completed, Asset Transferred).
  3. Click on a template to edit it, or click Add Template for a new event.
  4. Edit the Subject Line using placeholder variables (e.g., {{ticketNumber}}, {{assetName}}).
  5. Edit the Email Body using the rich text editor. Insert variables where dynamic data should appear.
  6. Use Preview to see how the email will look with sample data.
  7. Click Save to activate the template.
  8. Ensure SMTP is configured under System Config for emails to be sent.

Workflow 4: How to Configure Approval Thresholds

  1. Navigate to Administration > Master Settings in the sidebar.
  2. Go to the Approval Configuration section.
  3. Define the amount thresholds for each approval level:
  4. — HOD Level: Items up to this amount require only HOD approval.
  5. — Finance Level: Items between HOD threshold and this amount require Finance approval.
  6. — CEO/COO Level: Items above the Finance threshold require CEO/COO approval.
  7. Click Save to apply the new thresholds.
  8. All future approvals will follow the updated routing.

Workflow 5: How to Seed Default System Data

  1. Navigate to Administration > System Config in the sidebar.
  2. Look for the Seed Data section (typically available during initial setup).
  3. Select the data categories to seed: Departments, Asset Categories, Sub-Categories, Units of Measure, Priority Levels.
  4. Click Seed Default Data to populate the system with standard values.
  5. Review the seeded data under Master Settings and adjust as needed for your organization.
  6. Add your organization-specific departments, branches, and vendor records.
